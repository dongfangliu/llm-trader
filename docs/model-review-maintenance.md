# 模型复盘后台维护说明

## 状态流转

模型复盘后台仍复用旧的 `/api/admin/xbot/*` API 和 `xbot_predictions` 表。

1. 扫描候选：从 A 股和港股热度榜拉取标的，只进入临时候选列表，不写数据库。
2. 生成分析：维护者对候选逐条点击“生成分析”，系统调用 LLM 并写入 `pending` 记录。
3. 完整审核：进入 `/admin/model-review/:id` 检查方向、价格、目标日、摘要、四段分析和卡片预览。
4. 通过审核：`pending` -> `approved`，等待目标日结算。
5. 结算复盘：结算任务写入实际收盘、涨跌幅和命中结果，状态变为 `settled`。
6. 拒绝：`pending` 或 `approved` 可转为 `rejected`，不会公开展示。

## 日常操作

- 先点“扫描候选”，查看 A 股/港股数量、数据源状态、过滤数量和错误信息。
- 对合适标的点“生成分析”，生成后进入“待审核”。
- 默认逐条点“完整审核”，确认检查清单和 PNG 卡片正常后再通过。
- 到目标日收盘后点“结算”，结算成功的记录会出现在“已结算复盘”，并可打开公开页。
- 批量通过属于高级操作，只适合已经用其他方式完成审核的记录。

## 候选来源与代码规则

- A 股来源：AKShare EastMoney A 股热度榜。
- 港股来源：AKShare EastMoney 港股热度榜。
- A 股代码会归一化为 6 位纯数字：`SZ000066`、`SH601991`、`BJ430xxx`、`000001` 都可识别。
- 港股代码会归一化为 5 位数字：`HK00700`、`00700`、`700` 都会保存为 `00700`。
- 手动添加不会创建持久表记录，只加入当前候选列表；只有点击“生成分析”后才写入预测记录。
- 名称缺失时会先查 `symbol_names`，仍缺失则允许使用维护者填写的名称或代码。

## 常见失败原因

- `invalid_symbol`：数据源返回的代码格式无法识别，或手动输入的市场和代码不匹配。
- `below_min_price`：低于后台设置的最低价格过滤阈值。
- `st_or_delisted`：名称包含 ST、退市或退市风险特征。
- 数据源错误：候选区会显示 AKShare 抛出的错误信息。
- 分析生成失败：通常是行情数据缺失、LLM 配置错误、API 超时或额度不足。
- 卡片生成失败：检查前端服务 `/api/og/card`、字体文件和 `FRONTEND_URL` 配置。

## App Base URL 与字体

- PNG 卡片和公开复盘页脚网址优先读取后台系统设置里的 `email.app_base_url`。
- 旧设置 `xbot_product_url` 只作为兜底保留。
- 前端 Dockerfile 会下载 Noto Sans SC 的 400、700、900 OTF 字体；运行时也有 CDN 兜底。
- 如果中文粗体不正常，先检查 `frontend/public/fonts/NotoSansSC-*.otf` 是否存在。
