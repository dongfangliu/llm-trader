# 付费层级设计方案

> 版本：v1.1 | 目标市场：国内中小散户 | 主推渠道：小红书

---

## 一、目标用户画像

- **资金量**：4–8 万元
- **特征**：不懂技术分析，依赖直觉或跟风操作，急需简单可信的决策辅助
- **痛点**：
  - "这只股票我该不该买？"
  - "我现在被套了，该加仓还是割肉？"
  - "买多少合适？仓位怎么控？"
- **获客渠道**：小红书（图文笔记 + UGC 传播）

## 三、核心功能说明

### 3.1 深度研判（原四步决策，改名隐藏内部逻辑）

深度研判由 LLM 按四个维度依次分析，输出结构化结论：

| 步骤 | 名称 | 内容 |
|---|---|---|
| 第一步 | 🔍 市场诊断 | 市场状态、主要驱动因素、市场情绪、波动率 |
| 第二步 | 🎯 机会评估 | 机会类型、信号强度、确认程度、时间窗口、机会质量评级 A/B/C/D |
| 第三步 | ⚖️ 风险收益 | 目标价、止损价、风险收益比、主要风险因素 |
| 第四步 | 📋 执行方案 | 具体操作方向、仓位建议、入场时机、止损止盈价位 |

**免费版打码规则（让用户体验到厉害，但吊着升级）：**

| 字段 | 免费版处理 | 付费版 |
|---|---|---|
| `market_diagnosis`（第一步） | ✅ **完整显示** | 完整 |
| `opportunity_assessment`（第二步） | ✅ **显示机会方向+质量评级** | 完整 |
| `risk_analysis`（第三步） | ⚠️ 显示风险因素文字，**目标价/止损价数字打码** | 完整 |
| `execution_plan`（第四步） | ⚠️ 显示操作方向，**入场价/止盈止损全打码** | 完整 |
| `confidence`（置信度） | 🔒 打码 | 显示 |
| `reasons`（理由） | 显示前2条，**第3条打码** | 显示全部3条 |
| `target_price / stop_loss / take_profit` | 🔒 返回 null（后端处理） | 完整数字 |

> **设计原则**：市场诊断和机会评估充分展示专业度，建立信任；具体可操作的价格数字打码，用户拿不到直接操作依据，自然产生升级动机。

**免费版打码示例：**
```
📋 执行方案
操作方向：建议买入
入场价位：██ 元区间
止损位：██ 元    止盈目标：██ 元
仓位建议：██%
置信度：██%

理由：
① 均线系统发出黄金交叉信号
② 成交量同步放大，多头动能充足
③ ████████████████（升级解锁）

🔒 解锁完整深度研判 →
```

### 3.2 持仓参数智能分析

针对用户当前持仓状态，输入4个参数后给出个性化操作建议：

| 字段 | 含义 |
|---|---|
| 持有数量（股） | 当前持有的股数 |
| 成本价（元/股） | 买入均价 |
| 本股计划投入（元） | 对这只股票的总预算 |
| 最大持仓（股） | 允许持有的最大股数 |

- **规则**：4 项全填或全不填；全不填时按空仓处理
- **体验版**：不可用
- **标准版**：每日 1 次（用完后提示升级专业版）
- **专业版**：不限次数

> 这是最贴近 4–8 万散户痛点的功能，是从标准版升级到专业版的核心驱动力。

---

### 3.3 研判分享卡片（小红书传播钩子）

每次分析完成后，结果页右上角「📤 分享」按钮可生成分享卡片：
- **内容**：研判方向（买入/卖出/观望）+ 置信度 + 股票名称 + 分析摘要 + 品牌水印
- **生成方式**：纯 Canvas 绘制，直接下载为 PNG
- **所有层级均可用**（免费版置信度处显示 ██，形成升级暗示）
- 卡片顶部渐变色随研判方向变化（绿色=买入、红色=卖出、灰色=观望）

**核心裂变逻辑**：
```
用户获得分析结果 → 点击「分享」生成卡片 → 发到小红书/朋友圈/股友群
→ 好友看到结果感兴趣 → 扫码/链接体验 → 遇到打码 → 升级付费
```

> **规划中（未实现）**：标准/专业版专属主题色；卡片嵌入二维码

### 3.4 邀请奖励系统

- 每个注册用户自动生成唯一邀请码
- 被邀请人注册时填入邀请码 → **双方各获得 +10 次 bonus 分析次数**（永久累加）
- 与爱发电订阅系统完全解耦，独立计算
- 每日可用次数 = 层级基础次数 + 剩余 bonus 次数

## 四、定价策略

### 价格锚点

```
错误操作一次的成本：¥1,000–5,000
专业版 ¥49/月 = 每次分析仅 ¥3.3
标准版 ¥19.9/月 = 比一杯奶茶便宜
```

### 爱发电配置

| 套餐 | 爱发电金额 | DB tier |
|---|---|---|
| 标准版月付 | ¥19.9 | `basic` |
| 专业版月付 | ¥49 | `premium` |

- 金额 fallback 阈值：≥¥19.9 → basic，≥¥49 → premium
- **年付方案暂缓**，等月付用户基数稳定后扩展（需新建 Afdian 年付套餐 + 后端 timedelta(days=366)）

## 六、技术实现要点

### 后端（`backend/src/api/main.py`）
- `LIMITS` / `USER_LIMITS`：次数维持不变（free guest:2/day, free user:3/day, basic:5, premium:15）
- 打码逻辑：`_mask_result_for_free_tier()` 在 `/api/analyze` 响应构建处调用（服务端处理，非前端截断）
  - `target_price / stop_loss / confidence` 置 `null`
  - `risk_analysis` / `execution_plan` 文本中的数字用正则替换为 `██`
  - `reasons` 保留前2条，第3条替换为提示文字
  - 返回 `_masked: True` 标志位供前端渲染
- `User` 模型新增字段：`invite_code`、`bonus_quota`、`daily_position_usage`、`last_position_date`
- `UsageLog` 模型新增字段：`position_count`（追踪标准版持仓分析每日次数）
- 邀请接口：`POST /api/invite/use`，双方各 +10 `bonus_quota`
- Afdian 金额阈值：≥¥49 → premium，≥¥19.9 → basic

### 前端
- DB 枚举值 `free/basic/premium` 不变，仅改 UI 显示标签
- 深度研判对所有层级开放，免费版底部显示升级横幅，数字字段显示 `██`
- 持仓参数入口：`tier === 'basic' || tier === 'premium'`（含表单验证）
- 分享卡片：Canvas 原生绘制，点击下载 PNG（`page.tsx: generateShareCard()`）
- 邀请码 UI：账号页展示自己的邀请码 + 复制按钮；可输入好友邀请码兑换

### 数据库迁移
新字段通过 `_migrate_db()` 中的 `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` 语句自动补全，无需手动执行。
