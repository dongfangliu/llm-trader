## 预设配置详情

```json

  "bucket_size": 50,
  "lookback": 100,
  "threshold_multiplier": 2.5,
  "max_history": 1000
}
```
**适用场景：** 平衡各方面需求，适合大多数交易场景

**特点：**
- 适用性广
- 稳定可靠
- 推荐新手

---

### 短线交易
```json
{
  "bucket_size": 30,
  "lookback": 50,
  "threshold_multiplier": 2.0,
  "max_history": 800
}
```
**适用场景：** 日内短线交易，快速进出

**建议：**
- 适合波动频繁的时段
- 需要快速决策能力
- 注意过滤噪音信号

---

## 使用指南

### 快速开始（推荐新手）

1. 打开订单流Tab
2. 点击"参数配置"按钮
3. 选择"预设方案"标签页
4. 点击适合自己的预设配置卡片
5. 点击底部"保存配置"按钮

### 高级用法（经验用户）

1. 打开"预设方案"标签页
2. 选择一个接近需求的预设
3. 切换到"自定义"标签页
4. 微调参数（预设已填充）
5. 保存配置

### 场景切换

根据市场状态灵活切换配置：

**开盘阶段：**
```
使用"短线交易"或"高频交易"
- 波动大，机会多
- 需要快速反应
```

**盘中稳定阶段：**
```
使用"默认配置"或"趋势跟随"
- 趋势明确
- 稳定跟踪
```

**接近收盘：**
```
使用"保守策略"
- 降低风险
- 避免冲动
```

**主力拉升/派发：**
```
使用"大单监控"
- 追踪主力资金
- 识别关键时刻
```

## 错误排查

### 保存失败：undefined

**可能原因：**
1. 后端服务未启动
2. 订单流服务未初始化
3. 网络连接问题

**解决方法：**
```bash
# 1. 检查服务是否运行
curl http://localhost:8000/health

# 2. 查看浏览器控制台错误
# 打开开发者工具 -> Console 标签

# 3. 检查后端日志
# 查看 web_v2/server/logs/ 目录
```

### 参数范围错误

**错误示例：**
```
bucket_size 必须在 10-500 范围内，当前值: 1000
```

**解决方法：**
- 检查输入值是否在允许范围内
- 使用预设配置避免超出范围

### 配置不生效

**检查步骤：**
1. 确认点击了"保存配置"按钮
2. 查看是否有成功提示
3. 刷新配置查看当前值
4. 检查WebSocket连接状态

## API示例

### 获取当前配置
```bash
curl http://localhost:8000/api/order-flow/config
```

### 应用短线交易预设
```bash
curl -X POST http://localhost:8000/api/order-flow/config \
  -H "Content-Type: application/json" \
  -d '{
    "vpin": {"bucket_size": 30},
    "large_order": {
      "lookback": 50,
      "threshold_multiplier": 2.0
    },
    "orderbook": {"max_history": 800}
  }'
```

### Python脚本应用预设
```python
import requests

# 预设配置
PRESETS = {
    "scalping": {
        "vpin": {"bucket_size": 30},
        "large_order": {"lookback": 50, "threshold_multiplier": 2.0},
        "orderbook": {"max_history": 800}
    },
    "trend": {
        "vpin": {"bucket_size": 80},
        "large_order": {"lookback": 150, "threshold_multiplier": 3.5},
        "orderbook": {"max_history": 1500}
    }
}

# 应用配置
def apply_preset(preset_name):
    config = PRESETS.get(preset_name)
    if not config:
        print(f"预设 {preset_name} 不存在")
        return
    
    response = requests.post(
        "http://localhost:8000/api/order-flow/config",
        json=config
    )
    
    if response.ok:
        print(f"✅ 已应用 {preset_name} 配置")
        print(response.json())
    else:
        print(f"❌ 应用失败: {response.text}")

# 使用
apply_preset("scalping")  # 应用短线交易配置
```

## 最佳实践

### 1. 根据时段选择预设
```
09:30-10:00 早盘 → 短线交易/高频交易
10:00-14:30 盘中 → 默认配置/趋势跟随
14:30-15:00 尾盘 → 保守策略
```

### 2. 根据波动率调整
```
低波动 → 短线交易（提高敏感度）
正常波动 → 默认配置
高波动 → 保守策略（降低噪音）
```

### 3. 根据交易风格
```
激进 → 高频交易/短线交易
平衡 → 默认配置/趋势跟随
稳健 → 保守策略/大单监控
```

### 4. A/B测试
```
上午：使用配置A
下午：使用配置B
复盘：对比效果，选择最佳
```
