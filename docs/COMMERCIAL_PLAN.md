# AI 智能股票分析助手 - 商业化方案

## 一、项目概述

### 核心定位
- **产品名称**: AI 智能股票分析助手
- **目标用户**: 小红书散户（投资4-5万，预算敏感）
- **核心价值**: ¥9/月 = 一杯奶茶钱 = 永久 AI 股票顾问

### 商业模式
| 套餐 | 价格 | 次数 | 目标 |
|------|------|------|------|
| 免费版 | ¥0 | 每天 1 次 | 引流体验 |
| 基础版 | ¥9/月 | 每天 5 次 | **主要收入** |
| 高级版 | ¥19/月 | 每天 15 次 | 重度用户 |

### 核心设计原则
1. **单股票查询** - 每次分析消耗1次次数，会员制才有意义
2. **零门槛** - 不注册、不登录、不存储隐私
3. **专业感** - 显示技术术语，显得专业
4. **低价多销** - ¥9/月，小红书散户也愿意付

## 三、会员体系设计

### 套餐详情

| 套餐 | 价格 | 每日次数 | 市场 | 功能 | 目标用户 |
|------|------|----------|------|------|----------|
| 免费版 | ¥0 | 1次 | A股 | 基础分析 + 历史记录 | 引流 |
| 基础版 | ¥9/月 | 5次 | 全部 | 完整分析 + 历史记录 | **主要收入** |
| 高级版 | ¥19/月 | 15次 | 全部 | **并发多查询** + 优先通道 | 重度用户 |

### 功能说明

| 功能 | 免费版 | 基础版 | 高级版 |
|------|--------|--------|--------|
| 单股票分析 | ✅ | ✅ | ✅ |
| A股/港股/美股 | ❌ | ✅ | ✅ |
| 查询历史记录 | ✅ | ✅ | ✅ |
| 并发多查询 | ❌ | ❌ | ✅ |
| 优先响应 | ❌ | ❌ | ✅ |

### 每日次数限制

```
限制逻辑:
1. 每天 0 点重置次数（不是按月）
2. 免费版: 每天 1 次
3. 基础版: 每天 5 次
4. 高级版: 每天 15 次
5. 次数用完 → 引导升级

注意:
- 并发多查询时，5只股票 = 消耗5次
- 历史记录保存在浏览器本地，不占用次数
```

### 次数扣减流程

```
用户点击"开始分析"
        ↓
检查用户身份 (设备指纹/浏览器)
        ↓
查询今日已用次数
        ↓
次数 < 限制?
        ├─ 是 → 扣减1次，执行分析
        └─ 否 → 提示升级
```

### 防薅策略

| 策略 | 说明 |
|------|------|
| 设备指纹 | 限制同一设备每日使用次数 |
| IP 限流 | 同一 IP 每日请求上限 |
| 次数不过月 | 每天重置，不累积到下月 |
| 付费门槛 | ¥9/月过滤白嫖党 |

## 四、临时信息设计（不存储）

### 设计原则
- ❌ 不注册账号
- ❌ 不存储持仓数据
- ❌ 不需要登录
- ✅ 每次分析时临时输入
- ✅ 关闭页面后数据消失
- ✅ 纯游客也能用

### 信息字段

| 字段 | 说明 | 示例 |
|------|------|------|
| 持有数量 | 现在持有多少股 | 100股 |
| 成本价 | 买入时的价格 | 1700元 |
| 本股计划投入 | 这只股票上总投资 | 200000元 |
| 最大持仓 | 最多持有多少股 | 200股 |

### 自然语言输入

```
用户输入: "我持有100股，成本1700，计划投20万，最多买200股"

系统自动解析:
✓ 持有数量: 100股
✓ 成本价: 1700元
✓ 本股计划投入: 200000元
✓ 最大持仓: 200股
```

### 数据库设计（仅存次数）

```sql
-- 设备指纹表（仅记录使用次数）
CREATE TABLE usage_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    device_id TEXT NOT NULL,      -- 设备指纹
    date DATE NOT NULL,           -- 日期
    count INTEGER DEFAULT 0,      -- 今日已用次数
    subscription TEXT DEFAULT 'free', -- free/basic/premium
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(device_id, date)
);
```

**注意：仅存储设备ID和次数，不存储任何个人隐私数据。**

### 历史记录存储设计

```typescript
// 前端使用 localStorage 存储历史记录
interface HistoryItem {
    id: string;
    symbol: string;           // 股票代码
    name: string;             // 股票名称
    action: 'buy' | 'sell' | 'hold';  // 建议操作
    confidence: number;        // 置信度 0-100
    targetPrice: number;      // 目标价
    stopLoss: number;         // 止损价
    currentPrice: number;     // 当前价格
    timestamp: number;         // 时间戳
    holding?: {               // 用户提供的持仓信息
        quantity: number;
        costPrice: number;
        plannedInvestment: number;
        maxPosition: number;
    };
}

// 存储示例
const history: HistoryItem[] = [
    {
        id: 'uuid-1',
        symbol: '600519',
        name: '贵州茅台',
        action: 'buy',
        confidence: 85,
        targetPrice: 1850,
        stopLoss: 1720,
        currentPrice: 1780,
        timestamp: Date.now()
    }
];

// 存储到 localStorage
localStorage.setItem('analysis_history', JSON.stringify(history));
```

**历史记录设计原则：**
- 存储在浏览器本地（localStorage），不上传服务器
- 不泄露用户持仓信息
- 用户可自行清除
- 每次查询后自动追加
- 支持查看之前任意次分析结果

```

{
    "symbol": "600519",           -- 股票代码

响应:
{
    "success": true,
    "remaining": 4,               -- 剩余次数
    "result": {
        "action": "buy",         -- buy/sell/hold
        "confidence": 0.85,
        "target_price": 1850,
        "stop_loss": 1720,
        "reason": "...",

            "current_holding": 100,
            "suggested_action": "sell",
            "suggested_quantity": 50,
            "reason": "仓位已满85%"
        }
    }
}
```

POST /api/analyze/batch

## 十、风险与应对

| 风险 | 说明 | 应对 |
|------|------|------|
| API 成本 | Claude 涨价 | 动态调整套餐价格 |
| Ko-fi 审核 | 可能被封 | 准备备选支付 |
| AKShare 限速 | 爬虫被封 | 增加缓存/备选源 |
| 服务器攻击 | DDoS | 限流 + 防护 |
| 用户流失 | 白嫖后离开 | 引导升级 + 口碑 |

## 十二、总结

### 核心优势

| 优势 | 说明 |
|------|------|
| **零门槛** | 不注册不登录，游客也能用 |
| **隐私安全** | 不存储持仓，不泄露投资信息 |
| **低价策略** | ¥9/月，一杯奶茶钱 |
| **专业感** | 显示技术术语，显得专业 |
| **防薅羊毛** | 设备指纹+次数限制 |

### 预期效果

- 用户流失率降低 80%（零门槛）
- 付费转化率 5-10%
- 客服咨询减少 90%
- 10 个付费用户覆盖服务器成本
